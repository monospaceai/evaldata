## 0.4.0 - 2026-07-04
### Added
* SqliteAdapter: run evals against a local SQLite database via a new `sqlite` platform (in-process, no extra dependencies). Build one with `sqlite_platform(name, path)`; `evaldata doctor --sqlite` checks it.
* ExecutionAccuracy: the standard text-to-SQL execution-accuracy scorer — runs the gold query and the model's SQL and compares the result rows. Its row-order (`when_ordered`/`ignore`), duplicate-handling (`multiset`/`set`), and column-alignment (`by_position`/`by_value`) options reproduce the official Spider and BIRD execution-accuracy comparators, and `evaldata bench` applies the right one per benchmark.
* load_spider() and load_bird(): turn the Spider and BIRD benchmark datasets into runnable eval cases. Run a model over them with run_benchmark(...) or the `evaldata bench <spider|bird>` CLI to get an execution-accuracy number. Use SCHEMA_PROMPT_TEMPLATE to put the database schema in the prompt.
* `evaldata fetch <spider|bird>`: download, verify, and cache a benchmark dataset — pinned to a known SHA-256 with a case-count and SQLite integrity check — so `evaldata bench` can run against it. Downloads are explicit (never during a run).
* Evaluate AI SQL against a dbt project: `load_dbt(...)` reads a built project's `manifest.json` and `catalog.json` into a `DbtContext` — models, sources, resolved column types, and schema context — and builds text-to-SQL eval cases you score against gold queries on your own warehouse, no dbt Cloud account. `platform_from_profile` resolves the warehouse connection from a dbt profile, and `evaldata dbt-bench` runs a model over a cases file. Install the `dbt` extra.
* Evaluate dbt Semantic Layer (MetricFlow) queries: `load_dbt_metrics(...)` reads the semantic manifest's metrics and dimensions and builds metric-query eval cases. `MetricLayerSolver` asks a model to answer each question with a metric query, and `metric_layer_equivalence` scores it with a cascade — resolve-and-compare, run-and-compare, then an LLM judge — that runs the cheapest check first and stops at the first verdict; `MetricLayerJudge` is the judge tier on its own. `evaldata sl-bench` runs it end to end. Install the `dbt-sl` extra (`dbt-metricflow`).
### Changed
* Result schemas are now typed or untyped. `Schema` is a union of `TypedSchema` (columns with types) and `UntypedSchema` (column names only, returned by engines like SQLite whose driver reports no result-column types); type comparison is inconclusive rather than refuting when types are absent. Breaking: `evaldata.types.Schema` is now this union alias, not a class — construct `TypedSchema(...)` directly.

## 0.6.0 - 2026-07-09
### Added
* Snowflake Cortex Analyst solver: evaluate a deployed Cortex Analyst through the usual scoring cascade. `CortexAnalystSolver` sends a natural-language question to the Cortex Analyst REST endpoint (reusing your Snowflake connection's session token) and returns the SQL it generates, which evaldata runs and scores against a gold query. Install the `cortex` extra.
* Export the extension surface from the top-level `evaldata` namespace: the `Scorer` and `Solver` protocols, `EquivalenceCheck`, `QueryRunner`, `ScoreContext`, the platform-ref builders (`duckdb_platform`, `sqlite_platform`, `postgres_platform`, `databricks_platform`, `snowflake_platform`), and the LLM value types a custom backend returns (`Usage`, `Completion`, `TextCompletion`, `StubLlm`).
* Concurrent solver calls in benchmark runs: `run_benchmark` and `run_metric_benchmark` take `max_concurrency` (and `bench`/`dbt-bench`/`sl-bench` take `--max-concurrency`) to fan model calls out on a thread pool. Execution and scoring stay serial in case order, and the default of 1 keeps today's behavior.
### Changed
* `TypedSchema` and `UntypedSchema` now reject duplicate column names at construction with a validation error, instead of carrying them positionally. A query whose result set has duplicate output column names is still surfaced as a typed `duplicate_columns` execution error, not a crash.
* A scorer paired with the wrong `expected` kind (e.g. `ExecutionAccuracy` without a `GoldQuery`) now returns an inconclusive `ScoreResult` flagged `metadata["scorer_misconfigured"]` instead of raising `TypeError`, so the misconfiguration fails the case visibly rather than aborting the whole run.
### Fixed
* `ResultSetEquivalence` generated SQL that failed on Snowflake and SQLite: the keyless diff used `EXCEPT ALL` (unsupported on both) and keyed comparisons quoted internal aliases that Snowflake case-folds. The scorer now emits portable SQL, verified by conformance tests that run every scorer against every supported engine.
* The Snowflake examples seed into `SNOWFLAKE_DATABASE`/`SNOWFLAKE_SCHEMA` when set, instead of always requiring account-level `CREATE DATABASE` privilege.
